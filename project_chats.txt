TOPIC: Keyword mapping function — classify keywords in "Organic Keywords All & Keyword Mapping"

────────────────────────────────────────────────────────────────────────────────
CHANGES MADE
────────────────────────────────────────────────────────────────────────────────

1. NEW OPTION 10 — keyword_mapping()
   - Reads from Overview: building name (B53), city (B49), neighborhoods (B51),
     luxury mode flag (B55, YES/NO)
   - Classifies every keyword in "Organic Keywords All & Keyword Mapping" column A,
     writes result to column F ("Category"):
     · branded       — keyword contains full building name OR any significant token
                       from it (tokens < 4 chars and generic RE words filtered out)
     · primary       — keyword contains "apartments" + city → whole row green
                       (if B55=YES: also "apartments" + city + "luxury" → primary)
     · tracking only — keyword contains "apartments" + any neighborhood
     · secondary     — everything else
   - Green highlight applied via Sheets API batchUpdate:
     · One request resets all data rows to white (removes stale highlights)
     · Individual requests per primary row (small subset — efficient at scale)
   - Sheet auto-resized to 6 columns before writing F if needed (sheet was 5-col)
   - gspread update() call uses new API (values first, range second) to silence
     DeprecationWarning

2. SHEET RENAMED: "Organic Keywords All" → "Organic Keywords All & Keyword Mapping"
   - All 7 occurrences updated: worksheet(), add_worksheet(), print strings, comments

3. GUARD on semrush_data() top-5 block (B27:D31)
   - Added `if not all_rows: return` guard before the top-5 keywords section
   - If the keyword sheet is empty, B27:D31 is not touched (no clear, no write)

  Max columns: 5 → added kw_sheet.resize(cols=6) before writing to column F

TOPIC: Ahrefs & SEMrush API integration into main.py (SEO onboarding automation)

────────────────────────────────────────────────────────────────────────────────
CHANGES MADE
────────────────────────────────────────────────────────────────────────────────

1. ADDED import requests + Ahrefs/SEMrush constants
   - Added `import requests`
   - AHREFS_API_TOKEN, AHREFS_BASE_URL = "https://api.ahrefs.com/v3"
   - SEMRUSH_API_KEY, SEMRUSH_BASE_URL = "https://api.semrush.com/"

2. MENU OPTION 7 — ahrefs_data()
   - 3 API calls to Ahrefs v3 (Bearer token auth):
     · /site-explorer/domain-rating (mode=domain) → Ahrefs Rank
     · /site-explorer/domain-rating (mode=exact, target=homepage) → Homepage UR
     · /site-explorer/backlinks-stats → Referring domains count
     · /site-explorer/anchors → Top 5 anchors by refdomains
   - Discovered: Ahrefs API v3 requires Enterprise plan ($1,499/mo)
     → Current plan returns 403 "Insufficient plan" for real domains
     → Set target = "ahrefs.com" (TEST MODE) to verify code works
   - Base URL updated from apiv2.ahrefs.com (discontinued Nov 2025) to api.ahrefs.com/v3

3. MENU OPTION 8 — semrush_data()
   - Reads domain from Overview!B45 (GSC property dropdown), strips to clean domain
   - Reads city from Overview!B49 (mandatory)
   - Reads neighborhoods from Overview!B51 (mandatory, comma-separated list)
   - Fetches domain_organic from SEMrush API with pagination (10,000 rows/page)
     Columns: Ph (Keyword), Po (Position), Nq (Search Volume), Ur (URL)
   - Writes all keywords to new tab "Organic Keywords All"
   - Writes position counts to Overview:
     · B22 = keywords in positions 1–3
     · B23 = keywords in positions 4–10
     · B24 = keywords in positions 11–100
   - Writes top 5 keywords (pos 4–20) to Overview B27:D31:
     · Filter: keyword contains "apartments" AND (city OR any neighborhood)
     · Fallback: keyword contains "apartments" only (if no location match found)
     · Always batch_clear B27:D31 before writing to prevent stale data
     · Sorted by search volume descending

────────────────────────────────────────────────────────────────────────────────
KEY DECISIONS & NOTES
────────────────────────────────────────────────────────────────────────────────

- Ahrefs v2 was fully discontinued November 2025; all new code uses v3
- Ahrefs v3 site-explorer endpoints require Enterprise plan only
- SEMrush domain_organic works with current plan
- neighborhoods (B51) stored as Python list, split by comma
- "apartments" keyword filter is mandatory (no fallback removes it)
- batch_clear added to prevent old sheet data persisting when filter returns empty
